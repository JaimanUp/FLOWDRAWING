Development Roadmap
Flow-Guided Generative Drawing Engine

# Prototype Direction
- This document is targeted at a **quick prototype** using Java and Processing as originally proposed.
- Core subsystems (vector field, bots, brushes) will follow the designs described below without major deviations.
- Certain parameters (field normalization threshold, radar sample count) are exposed as configuration, allowing future tuning but not changing the fundamental architecture.
- Performance targets are aspirational; the initial implementation should focus on correctness and clarity rather than premature optimization.

Phase 2 — Vector Field Engine
    Goal: Build the mathematical core.

    Data Structure
        Grid of vectors:
            class VectorField {
                int cols
                int rows
                float cellSize

                PVector[][] vectors
            }
        Typical configuration:
            cellSize = 10 px
        For 4000px canvas:
            400 x 400 grid
            160,000 vectors

    Field Rules
        Each vector stores:
            vx
            vy
        Magnitude limit:
            maxMagnitude
        Global rule (VERY IMPORTANT):When 30% of vectors reach maxMagnitude:
            scaleFactor = maxMagnitude / currentMax
            scale all vectors
            
            This becomes a function:
                normalizeFieldIfNeeded()

Phase 3 — Field Visualization

            Allow toggle in UI.

Phase 4 — Brush System
    Goal: Allow user to draw flow fields.
    Tools
        Tool                 Falloff
        Pencil                 Linear
        Brush                   Gaussian
        Marker                  Hard Edge
    Brush Engine
        Formula:
            vector += mouse_delta \* falloff(distance)
        Where
             mouse_delta = currentMouse - previousMouse
    Falloff Functions
        Linear:
            1 - (distance/radius)
        Gaussian:
            exp(-(d² / sigma²))
        Hard Edge:
            if distance < radius → 1
            else → 0
    Brush Class
        class Brush {
            float size
            float hardness
            float spacing
            FalloffType falloff
            }

    Spacing behaves like Photoshop spacing.

Phase 5 — Bot Simulation Engine
    Goal: Generate trajectories.
    Bot Structure
        class Bot {

            PVector position
            PVector velocity

            float life
            float maxLife

            float radar
            float drift

            ArrayList<PVector> path
        }     
    Bot Spawn
        Random:
        position = random(canvas)
        velocity = randomDirection
        life = maxLife
    Motion Algorithm
        Each frame:
            fieldForce = sampleField(position)
            driftVector = randomWalk()
            direction = fieldForce + driftVector
            velocity = normalize(direction) \* speed
            position += velocity

    Field Sampling
        Use bilinear interpolation.
        Steps:
            1. Find cell indices
            2. Get 4 surrounding vectors
            3. Interpolate

Phase 6 — Radar Sampling Optimization
    Bots consider vectors inside radar.
    Instead of scanning all:
    Use 16 radial samples.
        for angle in 16 directions
            sample field
    This is fast and stable.

Phase 7 — Path Streaming System
    Paths grow continuously.
    Memory must remain safe.
    Each bot stores:
        ArrayList<PVector>
    But add point only if:
        distance(lastPoint, newPoint) > threshold
    Threshold:
        1–2 px
    This keeps paths compact.

Phase 8 — Adaptive Smoothness Controller
    Goal: prevent jitter.
    Compute:
        angleChange = angle(v_prev, v_current)
    Apply smoothing:
        v = lerp(v_prev, v_current, smoothingFactor)

Phase 9 — Stability Supervisor
    Monitor:
        fieldVariance
        botDensity
        curvatureVariance
    If unstable:
        reduce drift
        reduce magnitude

Phase 10 — Rendering Engine
    Draw bot paths.
        beginShape()
        for point in path
            vertex()
        endShape()

    Bots themselves can be drawn as:
        point
        circle
        triangle (direction indicator)

Phase 11 — Color System
    Color options:
        Mode 1 — constant
        Mode 2 — opacity based on life
        Mode 3 — global palette cycling

    Opacity curve:
        fade in
        mid life stable
        fade out
    Gaussian style.

Phase 12 — Export System
    Export formats:
        SVG
        DXF

Phase 13 — Path Optimization
    Two steps.
        Step 1 — Point Reduction
            Douglas-Peucker algorithm.
            Parameter:
                epsilon = slider(1–10)
        Step 2 — Travel Optimization
            Nearest neighbor ordering.
            Goal:
                minimize pen travel

Phase 14 — Topology Preservation
    Prevent bad simplifications.
    Check:
        segment angle
        segment length
        distance thresholds
    Protect loops and branches.

Phase 15 — Project Save System
    Store:
        canvas size

        seed
        background image
    JSON file.

Phase 16 — Performance Optimization

Roadmap Revision — 2026-04-02

        Based on the current implementation status, the next
        concrete development steps are:

        Short Term (prototype completeness)
                • Finalize brush + eraser tools
                        - Add explicit brush / eraser modes in the Sketch panel.
                        - Implement eraser behavior as vector subtraction toward zero.
                        - Verify spacing and falloff behavior (Linear / Gaussian / Hard Edge)
                            match the software requirements.

                • Implement radar-based bot sampling
                        - Use each bot's radar radius to aggregate multiple field samples
                            (radial or neighborhood sampling) instead of a single point.
                        - Apply distance-based dependency curves (Linear / Gaussian /
                            Hard Edge) when combining samples.
                        - Keep motion numerically stable and visually smooth.

                • Tie bot life to background
                        - Sample background luminance under each bot.
                        - Make life decay faster over light regions and slower over dark
                            regions, as defined in the requirements.

                • Improve simulation controls
                        - Add explicit start / pause / reset controls for the Botfield.
                        - Make it easy to edit the field, then run bots as a separate step.

        Medium Term (feature parity)
                • Expose field / grid parameters in UI
                        - Add controls for grid resolution and global magnitude scale in
                            the Field panel, wiring them into the VectorField.
                        - Expose field visibility / transparency per layer where useful.

                • Refine visualization & layer controls
                        - Polish arrow vs flow-map modes and ensure they perform well on
                            large canvases.
                        - Add per-layer opacity sliders (background / field / sketch /
                            bots) for better visual control.

                • Implement basic SVG export
                        - Convert bot paths (ArrayList<Vector2D>) into SVG polylines.
                        - Wire this to the existing "Export as SVG" menu entry.

        Later (robust tool)
                • Add project save / load skeleton
                        - Introduce a simple JSON-based project format (canvas size,
                            field data, bot settings, seed, background reference).
                        - Connect to the disabled Save / Open menu items.

                • Performance and stability pass
                        - Profile with a larger canvas and ~500 bots.
                        - Tune thresholds and caps (path point thresholds, max bots,
                            optional downsampled visualization).
                        - Add sanity checks for field normalization and bot density.
