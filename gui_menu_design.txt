# GUI Interface Plan - FlowDrawing

This document defines the general GUI structure before implementation. It is based on the current project roadmap and the existing codebase, which already includes canvas rendering, vector field controls, sketch controls, and bot simulation controls.

--------------------------------------------------

## 1. General Interface Structure

The application interface will be split into three main areas:

Approved shell decision:
- Keep the native Windows window bar and standard JFrame decorations.
- Do not build a custom integrated title bar.
- Use a standard Swing JMenuBar directly below the native window bar.
- Place the left vertical tool panel below the JMenuBar.
- Keep the canvas as the dominant central workspace.

1. Top window frame
- Native operating system window frame.
- Includes the standard window controls:
	- Minimize
	- Maximize / Restore
	- Close

2. Top application menu bar
- A standard horizontal menu integrated into the main screen frame.
- Main categories:
	- File
	- Edit
	- View
	- Help

3. Left vertical tool panel
- A second interface bar located on the left side of the window.
- This is the main working control area for the drawing system.
- It will be organized with four tabs at the top of the panel.
- Only one tab is active at a time.
- Each tab reveals a different set of controls.

4. Central workspace
- Main canvas area where background, vector field, sketch strokes, and botfield are rendered.
- Zoom, pan, painting, and simulation happen here.

--------------------------------------------------

Implementation decision:

- It is visually separate from the native title bar.
- This avoids custom window-chrome work and keeps normal Windows behavior intact.

## 3. Left Vertical Tool Panel (Tool-Centric Architecture)

This is the main control surface for the creative workflow. Uses a hierarchical tool-ribbon approach.

### Three-Level Structure

#### Level 1: Layer Tabs (Top - Fixed)
- BKGD (Background)
- VECFD (Vector Field)
- BOTFD (Bot Field)

**Behavior**:
- Layer tabs appear at the top of the left panel
- Clicking a tab switches the tool ribbon and settings below
- Only one layer tab active at a time
- Visually indicated: selected tab highlighted in neon green

#### Level 2: Tool Icon Ribbon (Fixed)
- First icon: **Visibility toggle** (☑/☐) - toggles layer visibility
- Remaining icons: Tool selection icons (paint, erase, place, etc.)
- Each layer has different tools

**Behavior**:
- Tools appear as small icon buttons (32-44px square)
- Visibility icon is always first in ribbon
- Clicking visibility toggles layer on/off (doesn't change tool)
- Clicking a tool icon selects that tool and updates settings below
- Selected tool highlighted in neon green
- Inactive tools shown in gray

#### Level 3: Settings Panel (Scrollable)
- Dynamically changes based on selected tool
- Always includes: **Transparency slider** (layer-level)
- Tool-specific controls below (size, hardness, strength, etc.)

**Behavior**:
- The panel remains fixed on the left side
- Contents update when tool is clicked (no expanding/collapsing)
- Scrollable if content exceeds panel height
- Scroll position resets per tool for consistency

## 4. Background Layer (BKGD)

Purpose:
- Manage the reference or visual background layer behind all simulation and drawing layers.

### Tool Ribbon

```
[ ☑ ] [ ● Color ] [ • Image ] [ • Clear ]
 ↑       ↑          ↑           ↑
 |       |          |           └─ Clear layer button
 |       |          └─ Load/manage background image
 |       └─ Select solid background color
 └─ Visibility toggle
```

### Tools & Settings

#### Color Tool (Selected by default)
- **Purpose**: Set background color
- **Settings**:
  - Transparency slider (layer-level)
  - Color picker button

#### Image Tool
- **Purpose**: Load and manage background image
- **Settings**:
  - Transparency slider (layer-level)
  - Load Image button
  - Clear Image button

#### Clear Tool
- **Purpose**: Clear layer
- **Settings**:
  - Confirmation message
  - Clear button

### Visibility Toggle Behavior
- Clicking eye icon (☑/☐) hides/shows background layer
- Doesn't affect which tool is selected
- Transparency still applies when layer is visible

## 5. Vector Field Layer (VECFD)

Purpose:
- Control how the vector field is displayed, visualized, and edited.

### Tool Ribbon

```
[ ☑ ] [ ● Paint ] [ • Erase ] [ • Overlay ] [ • Clean ] [ • Random ] [ • Reset ]
 ↑       ↑
 |       └─ Draw field forces with brush
 └─ Visibility toggle
```

### Tools & Settings

#### Paint Tool (Default)
- **Purpose**: Draw force vectors into the field
- **Settings**:
  - Transparency slider (layer-level)
  - Falloff Type toggle: Linear | Gaussian | Hard Edge
  - Size slider (brush size)
  - Hardness slider (falloff shape)
  - Strength slider (force magnitude)

#### Erase Tool
- **Purpose**: Remove/reduce field vectors
- **Settings**:
  - Transparency slider (layer-level)
  - Falloff Type toggle: Linear | Gaussian | Hard Edge
  - Size slider
  - Hardness slider
  - Strength slider

#### Overlay Tool
- **Purpose**: Fill region with field force
- **Settings**:
  - Transparency slider (layer-level)
  - Falloff Type toggle: Linear | Gaussian | Hard Edge
  - Size slider
  - Hardness slider
  - Strength slider

#### Clean Tool
- **Purpose**: Normalize/clean field without erasing
- **Settings**:
  - Transparency slider (layer-level)
  - Falloff Type toggle: Linear | Gaussian | Hard Edge
  - Size slider
  - Hardness slider
  - Strength slider

#### Random Tool
- **Purpose**: Generate randomized field
- **Settings**:
  - Transparency slider (layer-level)
  - Falloff Type toggle: Linear | Gaussian | Hard Edge
  - Size slider
  - Hardness slider
  - Strength slider

#### Reset Tool
- **Purpose**: Clear entire field to zero
- **Settings**:
  - Confirmation message
  - Reset button

### Visualization
- Arrow mode: Shows directional vectors
- Heatmap mode: Shows magnitude as color intensity
- Mode toggle in View menu (not in sidepanel)

### Visibility Toggle Behavior
- Clicking eye icon shows/hides field layer
- Tool selection remains unchanged
- Transparency still applies when field is visible

## 6. Bot Field Layer (BOTFD)

Purpose:
- Control bot-generated drawing layer and simulation behavior through various bot spawning and modification tools.

### Tool Ribbon

```
[ ☑ ] [ ● Place ] [ • Random ] [ • Auto ] [ • Clear ] [ • Export ]
 ↑      ↑
 |      └─ Place bots manually
 └─ Visibility toggle
```

### Tools & Settings

#### Place Tool (Default)
- **Purpose**: Manually place bots one at a time
- **Settings**:
  - Transparency slider (layer-level)
  - **General Settings**:
    - Amount slider (number of bots to place)
    - Color picker button
  - **Force Balance Settings**:
    - Vector Field influence slider
    - Repulsion influence slider
    - Drifting influence slider
  - **Life Settings**:
    - Duration slider (bot lifespan)
    - Velocity slider (movement speed)
    - Illuminance decay slider
  - **Field Radar Settings**:
    - Radar falloff type: Linear | Gaussian | Hard Edge
    - Radar radius slider
    - Hardness (center weight) slider
    - Strength slider
  - **Repulsion Radar Settings**:
    - Repulsion falloff type: Linear | Gaussian | Hard Edge
    - Repulsion radius slider
    - Hardness slider
    - Strength slider
  - **Drifting Settings**:
    - Drifting mode: Unique | Collective | Canvas
    - Velocity slider
    - Strength slider
  - [Show More ▼] for additional parameters

#### Random Tool
- **Purpose**: Randomly spawn bots across canvas
- **Settings**: Same as Place tool

#### Auto Tool
- **Purpose**: Continuously spawn bots automatically
- **Settings**:
  - Transparency slider (layer-level)
  - **Control**: Play | Pause buttons
  - Same settings as Place tool below

#### Clear Tool
- **Purpose**: Delete all bots or traces
- **Settings**:
  - Clear Bots button
  - Clear Traces button

#### Export Tool
- **Purpose**: Export bot paths and data
- **Settings**:
  - Export format selector
  - Export button

### Visibility Toggle Behavior
- Clicking eye icon shows/hides bot layer
- Tool selection remains unchanged
- Transparency still applies when layer is visible

## 7. Layout Behavior Guidelines

### Panel Structure (Top to Bottom)
1. **Layer Tabs [FIXED]** - Select which layer to work on
   - [●BKGD] [VECFD] [BOTFD]
   - Selected layer highlighted in neon green

2. **Tool Ribbon [FIXED]** - Icon-based tool selection
   - Visibility icon (☑/☐) always first
   - Tool selection icons follow
   - Selected tool highlighted in green
   - Inactive tools in gray

3. **Settings Panel [SCROLLABLE]** - Dynamic content
   - Updates based on selected tool
   - Always includes: Transparency slider
   - Tool-specific controls below
   - Scrolls vertically if content exceeds height

### Color & Style
- Background: Dark gray (#2E2E2E)
- Text: Off-white (#E8E8E8)
- Selected/Active: Neon green (#00FF00)
- Hover: Cyan (#00FFD1)
- Panel width: 230px default (200px minimum)
- All controls: 44px minimum height for WCAG compliance

### Interaction Model
- **Visibility toggle**: Separate from tool selection - doesn't change tool
- **Tool selection**: Click icon to switch tools - settings update instantly
- **Transparency**: Part of tool settings - each tool has own transparency value
- **Scrolling**: Smooth vertical scroll only, no horizontal scroll

### Architecture Principles
- Tool-centric design: Everything organized around available tools for each layer
- Settings-as-content: No collapsible/expandable groups - just clean settings
- Independence: Layer visibility is separate from tool selection
- Speed: All interactions are instant (no delays or animations)

## 9. Open Design Decisions

- **Keyboard shortcuts**: Should each tool have a hotkey (e.g., 'P' for Paint)?
- **Persistence**: Should transparency values per tool persist across sessions?
- **Undo/Redo**: Should tool settings changes be undoable?
- **Favorites**: Should users be able to save/recall tool presets?
- **Reset Button**: Should there be a "Reset All Settings" for current layer?

### Closed Decisions (Phase 2 UI Exploration)
- ✓ Tool-centric architecture (icon ribbons, not collapsible trees)
- ✓ Visibility as first icon in ribbon (not separate)
- ✓ Transparency in settings per tool (not per layer)
- ✓ No expanding/collapsing groups (all settings visible or scrollable)
- ✓ Layer tabs + tool icons + settings (3-level hierarchy)
- ✓ The application menu will not be integrated into the native title bar
- ✓ The window will keep the standard native minimize, maximize, and close controls
